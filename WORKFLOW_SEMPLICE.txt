## ✨ NOVITÀ: Workflow Unificato!

Ora hai UN SOLO workflow GitHub Actions che:
✅ Testa il build su OGNI push
✅ Deploya su GitHub Pages quando push su main
✅ Più semplice da capire e mantenere


┌─────────────────────────────────────────────────────────┐
│ 🎯 COME FUNZIONA ORA                                    │
└─────────────────────────────────────────────────────────┘

📝 FAI COMMIT SUL FEATURE BRANCH:
   └─> GitHub Actions: Build test ✅
   └─> Feedback immediato se ci sono errori
   └─> NESSUN deploy (tranquillo!)

🚀 FAI DEPLOY SU MAIN:
   └─> GitHub Actions: Build test ✅
   └─> GitHub Actions: Deploy su Pages 🌐
   └─> Sito aggiornato in 1-2 minuti!


┌─────────────────────────────────────────────────────────┐
│ 💻 USO PRATICO                                          │
└─────────────────────────────────────────────────────────┘

🔄 SVILUPPO QUOTIDIANO:
   ./quick-commit.sh
   → Test automatico su GitHub Actions
   → Deploy: NO (rispondi "n")

🌐 QUANDO SEI PRONTO A PUBBLICARE:
   ./quick-commit.sh "feat: ultima modifica"
   → Test automatico
   → Deploy: SI (rispondi "s")
   → Sito online in 1-2 min!


┌─────────────────────────────────────────────────────────┐
│ 📊 COSA VEDERE SU GITHUB                                │
└─────────────────────────────────────────────────────────┘

Actions Tab:
https://github.com/immacolato/red-tide/actions

Vedrai un solo workflow: "🚀 Build & Deploy"

Su feature branch:
  → Build test ✅ (no deploy)

Su main branch:
  → Build test ✅
  → Deploy ✅


┌─────────────────────────────────────────────────────────┐
│ 🎯 WORKFLOW COMPLETO                                    │
└─────────────────────────────────────────────────────────┘

1. Modifica file
   └─> code src/game.js

2. Commit interattivo
   └─> ./quick-commit.sh
   └─> GitHub testa automaticamente

3. Continua a lavorare...
   └─> Più commit sul feature branch
   └─> Ogni commit viene testato

4. Pronto? Deploya!
   └─> ./deploy.sh
   └─> O rispondi "s" a quick-commit
